make

Makefile

构建系统，依赖关系链

  • 文件越来越多时，一个个调用g++编译链会变得很麻烦
  • 于是，发明了make程序，只需要编写文件间的依赖关系，和生成各文件的规则
  • command：make a.out
  • 与用shell脚本直接写构建过程相比，make指明依赖关系的好处：
  1. 当更新 XXX.cpp 文件时只会编译XXX.o，而不需要把其他.cpp 文件重新编译
  2. 能自动并行地发起对.cpp的编译，加快编译速度 make -j
  3. 使用通配符批量生成构建规则，避免针对每个.cpp 与 .o 重复写g++(%o %.cpp)
  • 但坏处也很明显
  1. make在类Unix系统上是通用的，但在Windows上则不然
  2. 需要明确指明每个项目间的依赖关系，有头文件时特别头疼
  3. make语法特别简单，不像shell或python可以做很多判断
  4. 不同编译器有不同的flag规则，为g++准备的参数可能对msvc不适用
